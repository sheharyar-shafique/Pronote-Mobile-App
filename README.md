# Pronote Mobile App

iOS + Android build of [Pronote](https://github.com/sheharyar-shafique/Pronote-AI-Medical-Scribe) — an AI medical scribe for clinical voice-note capture, transcription, and template-driven documentation.

This repo is a **self-contained Capacitor + React app**. The same React code that runs on the web is packaged into a native iOS and Android app via Capacitor's WebView. Backend (Express + Supabase + OpenAI) is shared with the web app — see the main repo above.

```
Pronote-Mobile-App/
├── src/                    ← React app (pages, store, services, components)
│   └── native/             ← Capacitor bridge helpers (mic, storage, push, etc.)
├── public/
├── capacitor.config.ts     ← App ID com.pronote.clinical
├── platform-templates/     ← snippets to merge after `cap add ios|android`
│   ├── ios/Info.plist.snippet.xml
│   └── android/AndroidManifest.snippet.xml
├── ios/                    ← generated by `npm run add:ios` (macOS)
├── android/                ← generated by `npm run add:android`
└── package.json
```

## Quick start

### Prerequisites

| Target  | What you need                                                                |
| ------- | ---------------------------------------------------------------------------- |
| Both    | Node 18+, npm                                                                |
| iOS     | macOS, Xcode 15+, CocoaPods (`sudo gem install cocoapods`), Apple Developer  |
| Android | Android Studio (latest), Android SDK 34+, Java 17                            |

### Install

```bash
git clone https://github.com/sheharyar-shafique/Pronote-Mobile-App.git
cd Pronote-Mobile-App
npm install
cp .env.example .env       # then fill in VITE_API_URL etc.
```

### Run as a web preview (any OS)

```bash
npm run dev
```

Opens at `http://localhost:5173`. Use Chrome DevTools device-mode (`Ctrl+Shift+M`) to preview the mobile layout.

### Add the native projects

iOS (macOS only):

```bash
npm run add:ios
```

Android:

```bash
npm run add:android
```

These create the `ios/` and `android/` folders — full native projects you can open in Xcode and Android Studio.

### Apply the permission templates

After `add:ios` or `add:android` runs, **merge** these into the generated files:

| Template                                                            | Destination                                                |
| ------------------------------------------------------------------- | ---------------------------------------------------------- |
| `platform-templates/ios/Info.plist.snippet.xml`                     | `ios/App/App/Info.plist`                                   |
| `platform-templates/android/AndroidManifest.snippet.xml`            | `android/app/src/main/AndroidManifest.xml`                 |
| `platform-templates/android/data_extraction_rules.xml`              | `android/app/src/main/res/xml/data_extraction_rules.xml`   |

The Info.plist snippet contains the **mandatory** mic / camera / biometrics usage strings. **Apple will reject the build without them.**

## Daily build cycle

```bash
npm run build:mobile    # vite build  →  cap sync (copies into ios/ + android/)
npm run open:ios        # opens Xcode → Cmd+R to run on simulator/device
npm run open:android    # opens Android Studio → Run
```

`cap sync` also re-installs the native plugins listed in `package.json` — run it any time you add or remove a `@capacitor/*` plugin.

### Live reload during development

Point Capacitor at your local Vite dev server so saves reload the device:

```bash
# terminal 1
npm run dev   # http://localhost:5173

# in capacitor.config.ts, temporarily add a server.url block pointing
# to your dev machine's LAN IP, then:
npm run sync && npm run run:ios   # or run:android
```

Remove the `server.url` before producing a release build.

## Backend connectivity

This app talks to the Pronote backend (Express + Supabase + OpenAI). Set the API origin in `.env`:

```bash
VITE_API_URL=https://api.pronote.app   # production
# or for local dev:
VITE_API_URL=http://192.168.1.10:3001  # your dev machine's LAN IP (so a phone can reach it)
```

iOS simulator can use `localhost`. Android emulator must use `10.0.2.2` to reach the host.

### CORS

Capacitor's WebView sends `Origin: https://localhost`. Add `https://localhost` to the backend's `CORS_ORIGINS` env var.

## Releasing to the App Store

1. Open `ios/App/App.xcworkspace` in Xcode.
2. Set Bundle Identifier `com.pronote.clinical` and select your Apple Developer team.
3. Bump `CFBundleShortVersionString` and `CFBundleVersion`.
4. Add app icons to `ios/App/App/Assets.xcassets/AppIcon.appiconset/`.
5. Product → Archive → Distribute App → App Store Connect.
6. In App Store Connect:
   - Create the app (same bundle ID).
   - Fill out the **Privacy** section. For Pronote: Health & Fitness data → Yes; Contact info → Yes; Audio data → Yes.
   - Provide a privacy policy URL (required for any health app).
   - Provide a demo account so reviewers can log in.
7. Submit. First review on a health app typically takes 3–7 days.

### Apple's payment rule (READ BEFORE SHIPPING)

If users subscribe to Pronote *inside* the iOS app, Apple requires **In-App Purchase** (15–30% cut). Stripe/PayPal must be removed from the iOS build for digital subs.

Three options:
1. **Use Apple IAP** — cleanest approval; pay the cut.
2. **Hide upgrade UI on iOS, sell on web** — no cut, but Apple permits this only if you don't link/mention the website inside the app.
3. **External Link Account Entitlement** — possible for some B2B/health apps; takes weeks to approve.

## Releasing to Google Play

1. In Android Studio, set `applicationId = "com.pronote.clinical"` in `android/app/build.gradle`.
2. Generate an upload keystore (one-time):
   ```bash
   keytool -genkey -v -keystore pronote-upload.keystore \
     -alias pronote -keyalg RSA -keysize 2048 -validity 10000
   ```
   Store outside the repo. Add credentials to `android/keystore.properties` (gitignored).
3. Build → Generate Signed Bundle → Android App Bundle (`.aab`).
4. In Play Console: create app → Production track → upload `.aab` → fill **Data safety** form (declare PHI handling) → privacy policy URL → submit.

First review typically takes 1–3 days.

## HIPAA / privacy checklist

- [ ] Store JWTs via `src/native/storage.ts` (Preferences), not `localStorage`, on mobile.
- [ ] Disable iOS screenshot caching for screens with PHI.
- [ ] In Android `MainActivity`, add `WindowManager.LayoutParams.FLAG_SECURE` to block screenshots.
- [ ] Confirm `android:allowBackup="false"` (set via the Android snippet).
- [ ] Add a Face ID / fingerprint re-auth gate after backgrounding > 30s.
- [ ] BAA in place with Supabase + OpenAI (OpenAI requires Zero Data Retention for HIPAA).

## Native bridge helpers ([src/native/](src/native/))

Optional helpers that swap web APIs for native equivalents at runtime:

| File              | Purpose                                                                  |
| ----------------- | ------------------------------------------------------------------------ |
| `platform.ts`     | `isNative()`, `isIOS()`, `isAndroid()`                                   |
| `audio.ts`        | Native mic recording via `@capacitor-community/voice-recorder`           |
| `storage.ts`      | UserDefaults / SharedPreferences for sandboxed storage (use for JWTs)    |
| `app-shell.ts`    | Splash, status bar, keyboard insets, Android back button                 |
| `push.ts`         | APNs / FCM registration                                                  |

The web build short-circuits these via `isNative() === false`, so existing pages keep working unchanged.

## Troubleshooting

| Symptom | Likely fix |
| ------- | ---------- |
| `cap sync` fails: `webDir not found` | Run `npm run build` first — `cap sync` expects `dist/`. |
| iOS build: "Signing requires a development team" | Open `ios/App/App.xcworkspace`, App target → Signing & Capabilities → pick your team. |
| Android build: `SDK location not found` | Create `android/local.properties` with `sdk.dir=/path/to/Android/Sdk`. |
| Mic prompt doesn't appear on iOS | `Info.plist` is missing `NSMicrophoneUsageDescription` — apply the snippet. |
| App calls backend but gets CORS error | Add `https://localhost` to backend `CORS_ORIGINS`. |
| White screen on launch | Check WebView console (Safari → Develop → Simulator on iOS; `chrome://inspect` on Android). Usually a missing `webDir` or wrong API URL. |
